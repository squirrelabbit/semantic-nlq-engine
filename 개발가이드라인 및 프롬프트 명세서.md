제안서의 기술 부문이나 포트폴리오의 아키텍처 설계 섹션에 바로 활용하실 수 있도록, **[행정 지능형 인구 분석 플랫폼]**의 상세 구축 스펙과 기술 아키텍처를 엔지니어링 관점에서 집대성하여 정리해 드립니다.

이 내용은 개발자로서 시스템의 **확장성, 보안성, 그리고 데이터 무결성**을 어떻게 확보했는지 증명하는 데 초점이 맞춰져 있습니다.

### 1. 전체 시스템 아키텍처 (Full-Stack Architecture)

이 시스템은 **"데이터 저장 - 지식 정의 - AI 추론 - 서비스 제공"**의 4계층으로 분리된 **'Decoupled Agentic Architecture'**를 따릅니다.

#### **Layer 1: Data Infrastructure (저장 및 연산)**

* **DBMS:** PostgreSQL 16 (최신 파티셔닝 및 성능 최적화 활용)
* **Spatial Engine:** PostGIS (P-Cell 격자 데이터 및 행정동 경계 공간 연산)
* **Vector Engine:** pgvector (지역 지식 카드 및 보고서 템플릿 검색용 RAG 환경)
* **Data Pipeline:** Python 기반 ETL 프로세스 (SKT 데이터 및 공공 API 자동 적재)

#### **Layer 2: Semantic & Knowledge Layer (지식 정의)**

* **Metadata Registry:** 테이블 스키마, 조인 경로, 비즈니스 컬럼 정의를 JSON/Config 형태로 관리.
* **Administrative Logic Engine:** 행정표준코드(KIKmix)의 분동/말소 이력을 쿼리에 자동 주입하는 매핑 테이블.
* **Knowledge Store:** 지역적 특성(Knowledge Cards) 및 도메인 전문 용어 사전.

#### **Layer 3: AI Agent Orchestration (추론 및 실행)**

* **Core LLM:** Claude 3.5 Sonnet (복잡한 추론 및 SQL 생성 능력 탁월) 또는 GPT-4o.
* **MCP Server:** Model Context Protocol을 활용하여 LLM과 DB 간의 보안 통제 통로 구축.
* **Multi-Agent Workflow:**
* **Planner:** 질문 의도 파악 및 데이터 가용성 체크.
* **Coder:** 시맨틱 규칙을 준수하는 Text-to-SQL 생성.
* **Executor:** 생성된 SQL의 문법 검증 및 실행.
* **Interpreter:** 결과 데이터에 배경 지식을 결합하여 인사이트 도출.



#### **Layer 4: Service & UX Layer (인터페이스)**

* **Frontend:** React 18, Tailwind CSS, Lucide Icons.
* **Visualization:** Recharts 또는 D3.js (동적 데이터 시각화).
* **Push Engine:** 새벽 배치 작업을 통한 데일리 브리핑 생성 및 외부 메신저 API 연동.

### 3. 핵심 엔진 설계 (Key Engineering Logic)

#### **① Semantic-Driven SQL Generation (Text-to-SQL)**

단순히 질문을 SQL로 바꾸는 것이 아니라, 시맨틱 레이어를 거치며 **'행정적 제약 조건'**이 자동으로 추가됩니다.

* **Logic:** `User Question` → `Planner (Search Metadata)` → `Coder (Inject Logic: WHERE abolished_at IS NULL)` → `Final SQL`
* **효과:** 개발자가 모든 케이스의 SQL을 미리 짤 필요 없이, 시맨틱 정의만으로 무한한 분석 조합 생성 가능.

#### **② Dynamic Drill-down 시나리오 엔진**

분석 단계(유입지 → 연령 → 주변시설)를 상태 머신(State Machine)으로 관리합니다.

* **Logic:** 이전 단계의 분석 결과(JSON)를 다음 단계의 `Context`로 전달하여, AI가 "앞서 분석한 성남시 20대를 기준으로..."라고 인지하게 함.
* **효과:** 대화의 맥락이 끊기지 않는 심층적 원인 분석 가능.

#### **③ KIKmix 이력 보정 알고리즘**

과거 데이터 조회 시 현재 행정 구역과 매핑되지 않는 문제를 해결합니다.

* **Logic:** 쿼리 실행 전, `std_ymd`를 기준으로 해당 시점의 유효 코드를 조회하거나, 현재 코드로 환산된 집계 테이블을 우선 참조하도록 Planner 가이드.

### 5. 프로젝트 아키텍처의 의의 (포트폴리오 결론용)

1. **Agile Data Governance:** 행정 구역 변경 등 외부 환경 변화에 대해 시스템 소스 코드 수정 없이 메타데이터 업데이트만으로 즉각 대응 가능한 구조 설계.
2. **Human-AI Collaboration:** AI는 복잡한 데이터 연산과 초안 작성을 수행하고, 공무원은 정책 판단에만 집중할 수 있는 **'AI-Augmented Administration'** 실현.
3. **Scalable Intelligence:** MCP와 시맨틱 레이어를 통해 인구 데이터뿐만 아니라 경기도의 모든 행정 데이터로 확장 가능한 **범용 AI 분석 프레임워크** 구축.
