# 범용 시맨틱 NLQ 분석 엔진 PoC

## 프로젝트 개요
본 PoC는 **'Semantic Layer 기반 NLQ 분석 엔진'**의 기술적 완결성을 검증하고,
어떤 정형 데이터(SQL) 환경에서도 자연어로 데이터를 안전하고 재현 가능하게 분석할 수 있는
범용 엔진의 프로토타입을 확보하는 것을 목표로 한다.

## PoC 범위 (현실적으로 딱 이만큼)
### 데이터
- 핵심 데이터 1~2종 (예: 사내 샘플 데이터 또는 [공공데이터포털](https://www.data.go.kr/)의 공개 데이터)
- 지역 1~2곳
- 기간 제한 허용

### 기능
- 일일 자동 보고서 1종
- 심층분석 버튼 2~3개
- 질문 세트 고정
- Semantic Layer 최소 세트

### 운영
- 수동 배치 허용
- 관리자 화면 없어도 OK
- 일부는 스크립트 기반으로 운영

## PoC 고정 템플릿
### 일일 인구 자동 분석 리포트 (고정 템플릿)
이 구조는 PoC와 본사업 모두 동일하게 유지한다.

1. 오늘 요약 (고정, 2~3문장)
   - 전일 대비 전체 유입 인구 변화
   - 가장 큰 변화 지역 1~2곳
   - 특이사항 여부
2. 주요 변화 지역 (고정)
   - 증가 상위 3개 지역
   - 감소 상위 3개 지역
   - 표기: 지역명 / 증감률 / 전일 대비
3. 이상 징후 감지 (고정)
   - 기준 초과 지역 요약
   - 없을 경우: "특이사항 없음"
4. 참고 맥락 (조건부)
   - 축제 / 날씨 / 교통 이벤트 중 관련 항목만
5. 데이터 상태 (고정)
   - 기준일
   - 행정 코드 보정 여부
   - PII 마스킹 적용 여부

### 심층분석 버튼 = 질문 템플릿 (고정)
보고서 템플릿이 아니라 **질문 템플릿만 고정**한다.

- 버튼 1. 원인 탐색  
  [심층분석] 최근 변화는 단기 이벤트인가, 추세 변화인가?
  - 분석 대상: 선택 지역
  - 비교 기준: 최근 4주 평균 / 전년 동기
  - 출력: 비교 요약 + 해석

- 버튼 2. 장기 비교  
  [심층분석] 최근 3년 동일 기간과 비교
  - 분석 대상: 선택 지역
  - 기준: 연도별 평균
  - 출력: 추세 설명 + 현재 위치

- 버튼 3. 공간 확장 (선택)  
  [심층분석] 인접 지역과의 이동 패턴 비교
  - 분석 대상: 선택 지역 + 인접 지역
  - 출력: 차이 요약

LLM 출력 문단 구성/길이는 자율이며, 질문과 비교 기준만 고정한다.
이 템플릿은 PoC의 완성도를 높이기 위한 것이 아니라, PoC의 경계선을 고정하기 위한 것이다.

## PoC 성공 기준
1. 시맨틱 제약을 준수한 NLQ → SQL 변환이 안정적으로 수행된다.
2. 동일 질문에 대해 재현 가능한 결과가 생성된다.
3. 행정 코드 보정 및 PII 마스킹이 자동 적용된다.
4. 위험한 질문 또는 비허용 요청이 시스템적으로 차단된다.

## 데모 시나리오 (고정)
1. **기본 분석 데모**
   - 질문: “전일 대비 판교 유입 인구 변화”
   - 확인 포인트: 테이블 선택, 시맨틱 매핑, 마스킹
2. **비허용 질문 차단**
   - 질문: “개인 이동 경로 조회”
   - 확인 포인트: 정책 필터링 및 안전 메시지
3. **비교형 분석**
   - 질문: “최근 3년 대비 이례적인가?”
   - 확인 포인트: 비교 기준 고정, 일관된 해석
